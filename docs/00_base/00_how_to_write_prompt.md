# Prompt Writer Prompt

## 你的任務

使用者希望你把目前對話中討論的內容整理成一個可重複使用的 prompt 檔案，存放在 `/Users/chouwilliam/Medina/hub/prompts/` 目錄下。這樣未來使用者遇到類似的工作流程時，只要讓 AI 讀取該 prompt 檔案，就能快速進入狀態並執行相同的操作。

## 撰寫 Prompt 的規則

在撰寫 prompt 檔案時，你必須遵守以下規則：

第一，使用完整的文字段落來描述每一個概念或步驟，不要使用 bullet point 或簡短的條列式說明。原因是 AI 在讀取 prompt 時，完整的段落敘述能提供更多上下文，讓 AI 更容易理解整個工作流程的來龍去脈。簡短的 bullet point 雖然對人類閱讀很方便，但對 AI 來說反而會造成資訊不連貫、難以推理的問題。

第二，在 prompt 的開頭要有清楚的背景說明，解釋這個 prompt 是用來做什麼的、為什麼需要這個工作流程、以及相關的系統架構或歷史背景。這些背景資訊能幫助 AI 理解整個情境，而不只是機械式地執行指令。

第三，如果這個工作流程需要讀取特定的檔案，必須列出完整的絕對路徑，並且解釋每個檔案的用途和內容。不要只列出路徑，要說明為什麼需要讀這個檔案、讀完之後要注意什麼。

第四，描述操作步驟時，要按照邏輯順序一步一步說明，並且解釋每一步的目的。如果某些步驟之間有依賴關係，要明確指出先後順序和原因。

第五，如果有命名規則、格式要求、或其他慣例，要詳細說明這些規則的內容和背後的原因。這能幫助 AI 在執行時做出符合專案風格的決策。

第六，在 prompt 的結尾，要說明讀完這個 prompt 之後 AI 應該做什麼，例如詢問使用者要進行什麼操作、或是直接開始執行某個任務。

## 檔案命名規則

prompt 檔案的命名應該使用小寫英文字母和底線，名稱要能清楚表達這個 prompt 的用途。例如 pipeline_manager.md 是用來管理排程任務的，prompt_writer.md 是用來撰寫新 prompt 的。檔案格式統一使用 markdown。

## 執行流程

首先，回顧目前對話中討論的內容，整理出這個工作流程的核心目的、涉及的檔案、操作步驟、以及注意事項。

接著，詢問使用者這個 prompt 要取什麼名稱，或是根據內容建議一個適當的名稱。

然後，根據上述的撰寫規則，用完整的文字段落撰寫 prompt 內容。

最後，將完成的 prompt 寫入 `/Users/chouwilliam/Medina/hub/prompts/` 目錄下，並告訴使用者未來如何使用這個 prompt。
